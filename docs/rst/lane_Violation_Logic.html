lane_Violation_Logic.checkCrossedLaneLineToOtherSide(obj, lane, vehicle, much_area_on)¶
Return type

True if vehicle has retrogressed being near to the lane lines.

lane_Violation_Logic.checkRetrogress(obj, vehicle, where_is_vehicle)¶

this function checks retrogress with the help of average areas of first 5 and previous 5 frames. A retrogressing object ought to show opposite characteristics of area increase or decrease according to the lane it is in.

Parameters

  • obj – Lane Violation Object

  • vehicle – Vehicle Object

  • where_is_vehicle – the current location of the vehicle

Returns

True on retrogress, False otherwise.

lane_Violation_Logic.checkToporBottomNear(lane, vehicle)¶

check whether top part is near to the vehicle relative to its current location, or the bottom part. :param lane: the current active lane for vehicle :param vehicle: vehicle object :return: Top or Bottom string based on it’s location.

lane_Violation_Logic.checkVehicleLocation(lane_areas, vehicle)¶

Returns the location of vehicle by checking the bounding box and comparing with the lane areas. :param lane_areas: separated lane areas :param vehicle: vehicle object to check of :return: string of its location

lane_Violation_Logic.extendLaneLineUptoVehicle(obj, lane, extend_from, upto, up=True)¶

Extends Lane Line Upto Vehicle to detect if the current bounding box of the vehicle intersects the lane line or not.

Parameters

  • obj – detection object (Lane Violation Object)

  • lane – which lane line to extend lane line upto vehicle on

  • extend_from – from which part to extend lane line is passed from here

  • upto – upto which part to extend lane line upto is mentioned here.

  • up – Is the line to be extended upwards or downwards.

Returns

tuple of the location of lane line with respect to vehicle’s y otherwise None, None
